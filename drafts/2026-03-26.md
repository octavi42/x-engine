## 5. Thread hook + outline

> took HistorAI, renamed it to Clio, and shipped a browser version in one day. same ElevenLabs voice agent, same Firecrawl research pipeline, running entirely client-side. here's how the web architecture differs from mobile:

**Type:** thread hook
**Characters:** 222

**Thread outline:**
1. Hook (above)
2. Why rename: HistorAI was too generic. Clio is the Greek muse of history — fits a narrator app perfectly. also shorter, more brandable.
3. Mobile stack: React Native + Expo, native GPS via expo-location, ElevenLabs SDK with LiveKit WebRTC, Overpass API for landmark pins.
4. Web stack: Vite + React, browser Geolocation API, ElevenLabs WebSocket agent (no LiveKit needed in browser), Mapbox GL JS for map rendering.
5. Key difference: mobile uses a server-side conversation token flow. web connects directly via WebSocket — simpler auth, faster cold start.
6. Deployment: Vercel for frontend, Elastic Beanstalk for the API. had to add vercel.json rewrites to proxy /api/* and dodge mixed content blocks.
7. Shipped search bar with Mapbox Geocoding so you can narrate any location without physically being there. killer for the web demo.
8. Result: same AI narrator experience across mobile and browser. total build time: one day with Claude Code.
