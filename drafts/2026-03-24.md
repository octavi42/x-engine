## 5. Thread Hook + Outline

> you don't need to eject from Expo to write native iOS code. I built a custom blur module in 40 lines of Swift. here's the full process:

**Type:** thread hook
**Characters:** 135

**Thread outline:**
1. Hook (above)
2. The problem: needed native UIVisualEffectView blur for glass UI in HistorAI — no existing React Native library matched the design
3. Expo Modules API: run `npx create-expo-module`, write a Swift view that subclasses ExpoView, expose it to JS
4. The Swift: add UIVisualEffectView as subview, configure blur style (.systemUltraThinMaterial) and corner radius — that's the whole native layer
5. React Native side: import as a native component, use it like any View with style props. wrap buttons, headers, whatever needs blur
6. Result: glass blur buttons across the entire app. no eject, no Cocoapods issues, no third-party dependency to maintain
7. Rule of thumb: if the native code is under 50 lines, write the module yourself instead of adding a dependency
